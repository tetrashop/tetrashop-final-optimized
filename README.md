# 🛍️ Tetrashop — فروشگاه تعاملی تک‌صفحه‌ای (SPA)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Tests](https://img.shields.io/badge/tests-3%20passed-brightgreen)]()
[![Deploy with Vercel](https://img.shields.io/badge/deploy-Vercel-black?logo=vercel)]()

یک اپلیکیشن وب **ایستا** و **تعاملی** که فروشگاهی ساده با ۳ محصول دیجیتال را شبیه‌سازی می‌کند. کاربران می‌توانند محصولات را به سبد خرید اضافه یا از آن حذف کنند، امتیاز جمع‌آوری نمایند و خرید خود را نهایی کنند. تمام داده‌ها در **Local Storage** مرورگر ذخیره می‌شوند و با هر تعامل، امتیاز کاربر به‌روز می‌شود.

هدف از این پروژه نمایش یک **MVP فروشگاهی** بدون نیاز به سرور، با تمرکز بر:
- معماری فرانت‌اند ماجولار
- مدیریت state سمت کاربر
- تست‌پذیری
- طراحی واکنش‌گرا (Responsive)
- آمادگی برای استقرار روی سکوهای ابری (Vercel)

## ✨ ویژگی‌ها

- 🛒 **سبد خرید کاملاً پویا**  
  افزودن محصول، افزایش تعداد، حذف تک‌آیتم و محاسبهٔ خودکار مجموع قیمت.
- ⭐ **سیستم امتیازدهی گیمیفیکیشن**  
  هر افزودن (+۱۰)، حذف (-۵) و تکمیل خرید (+۵۰) امتیاز کاربر را تغییر می‌دهد.
- 💾 **ذخیره‌سازی پایدار در Local Storage**  
  سبد خرید و امتیاز حتی پس از بستن مرورگر حفظ می‌شود.
- 📱 **واکنش‌گرایی کامل (RWD)**  
  طراحی مدرن با CSS Grid و Flexbox که در تمام اندازه‌های صفحه نمایش زیبا می‌ماند.
- 🧪 **تست‌های خودکار**  
  سه تست واحد با Jest برای اطمینان از وجود فایل‌های اصلی و وجود Media Queryها.
- ☁️ **آماده برای Vercel**  
  فایل پیکربندی `vercel.json` سرو فایل‌های استاتیک را مدیریت می‌کند.

## 🚀 راه‌اندازی و اجرای محلی

### پیش‌نیازها
- [Node.js](https://nodejs.org/) (نسخهٔ ۱۴ به بالا)
- npm (همراه Node.js نصب می‌شود)

### مراحل

1. **مخزن را کلون کنید**
   ```bash
   git clone https://github.com/tetrashop/tetrashop-final-optimized.git
   cd tetrashop-final-optimized
```

1. وابستگی‌های توسعه را نصب کنید (فقط برای تست)
   ```bash
   npm install
   ```
2. اجرای پروژه
   از آنجایی که پروژه کاملاً ایستاست، می‌توانید فایل public/index.html را مستقیماً در مرورگر باز کنید.
      اما برای تجربهٔ بهتر (و جلوگیری از مشکلات CORS در صورت گسترش) از یک سرور محلی استفاده کنید:
   ```bash
   npx serve public
   ```
   سپس در مرورگر آدرس http://localhost:3000 را باز کنید.

🧪 اجرای تست‌ها

برای اجرای تست‌های خودکار (بررسی وجود فایل‌ها و صحت ساختار CSS):

```bash
npm test
```

خروجی مشابه زیر خواهد بود:

```
 PASS  test/app.test.js
  ✓ فایل index.html باید وجود داشته باشد
  ✓ فایل app.js نباید خالی باشد
  ✓ CSS باید حاوی استایل‌های ریسپانسیو باشد
```

🌐 استقرار روی Vercel

پروژه از پیش برای Vercel پیکربندی شده است. کافی است:

1. به Vercel.com بروید و با حساب GitHub خود وارد شوید.
2. روی New Project کلیک کنید و مخزن tetrashop-final-optimized را انتخاب نمایید.
3. Vercel به صورت خودکار vercel.json را تشخیص می‌دهد و فایل‌های داخل public را به عنوان ریشهٔ سایت تنظیم می‌کند.
4. روی Deploy کلیک کنید. تا چند ثانیه دیگر فروشگاه شما آنلاین خواهد بود.

همچنین می‌توانید با Vercel CLI نیز دیپلوی کنید:

```bash
npm i -g vercel
vercel login
vercel --prod
```

🧠 معماری و الگوهای طراحی

۱. الگوی ماژولار بدون فریم‌ورک

کل برنامه در یک فایل JavaScript سازماندهی شده اما توابع آن از اصل Single Responsibility پیروی می‌کنند:

· renderProducts() – رندر کارت محصولات
· renderCart() – رندر سبد خرید
· addToCart() – منطق افزودن
· updateScore() – مدیریت امتیاز
· saveCart() / loadCart() – ذخیره و بازیابی حالت

۲. State Management سمت کاربر

حالت برنامه (سبد خرید، امتیاز) در آبجکت‌های سادهٔ JavaScript نگهداری می‌شود و پس از هر تغییر در LocalStorage ذخیره می‌گردد. این روش برای MVPها الگوی Flux-like ساده‌شده محسوب می‌شود.

۳. بهبود تجربه کاربری (UX افکت‌ها)

هنگام افزودن محصول، متن دکمه برای ۸۰۰ میلی‌ثانیه به «✓ افزوده شد» تغییر می‌کند — بازخورد بصری فوری بدون نیاز به کتابخانهٔ خارجی.

📊 تحلیل دسترسی‌پذیری (Accessibility)

· استفاده از تگ‌های معنایی HTML5 (<header>, <main>, <footer>, <aside>).
· نسبت کنتراست رنگ‌های متن و پس‌زمینه طبق استاندارد WCAG 2.0 AA بررسی شده (متن سفید روی گرادینت بنفش / دکمه‌های سبز با متن روشن).
· دکمه‌ها دارای متن خوانا و قابل فهم هستند.
· پشتیبانی از جهت RTL (راست‌به‌چپ) برای زبان فارسی.

🔮 مسیر توسعهٔ آینده

· افزودن پشتیبانی از IndexedDB برای ذخیره‌سازی حجیم‌تر و آفلاین.
· پیاده‌سازی Service Worker و تبدیل به PWA کامل.
· تست‌های E2E با Cypress یا Playwright.
· دریافت پویای محصولات از یک API خارجی (مثلاً FakeStoreAPI).
· پنل مدیریت ساده برای افزودن/ویرایش محصولات.
